# Data Persistence

## Saved Data (backend/data/)

| File | Size | Description |
|------|------|-------------|
| `sp100_daily_prices.csv` | ~11 MB | S&P 100 daily closes (Yahoo Finance) |
| `news_sentiment_updated.json` | ~101 MB | News articles with FinBERT sentiment |
| `deepvar_results/` | ~7 MB | DeepVaR model and results |
| `var_cache/` | ~890 MB | VaR time-series cache |
| `market_caps.json` | small | Per-ticker USD market cap (Finnhub + Yahoo fallback) |

## Persistence

- **Docker**: `./backend/data` is mounted to `/app/data` — data persists across container restarts.
- **Git**: `backend/data/` is gitignored — data is not committed.

## Backup

To backup before major changes:

```bash
BACKUP="backend/data_backup_$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP"
cp backend/data/sp100_daily_prices.csv "$BACKUP/"
cp backend/data/news_sentiment_updated.json "$BACKUP/"
cp -r backend/data/deepvar_results "$BACKUP/"
cp backend/data/market_caps.json "$BACKUP/" 2>/dev/null || true
```

## Restore from Backup

```bash
cp backend/data_backup_YYYYMMDD_HHMMSS/sp100_daily_prices.csv backend/data/
cp backend/data_backup_YYYYMMDD_HHMMSS/news_sentiment_updated.json backend/data/
cp -r backend/data_backup_YYYYMMDD_HHMMSS/deepvar_results backend/data/
cp backend/data_backup_YYYYMMDD_HHMMSS/market_caps.json backend/data/ 2>/dev/null || true
```

## Market caps

- Generated by `data_updater.update_market_caps()` (also runs at the end of `run_update`).
- Standalone: `python backend/data_updater.py --market-cap-only` (uses `FINNHUB_API_KEY` when set, else Yahoo only).
